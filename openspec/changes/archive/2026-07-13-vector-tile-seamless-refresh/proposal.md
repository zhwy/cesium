## Why

当前矢量瓦片图层在运行时更新样式或切换显隐状态时，会同时失效四叉树瓦片并清空已提交渲染集，导致新瓦片完成请求、解码、建桶和 GPU 预热之前出现空白帧。需要在保证新样式和最新瓦片数据确实重建的同时，保留可用旧画面作为过渡，从而消除可见闪烁。

## What Changes

- 为矢量瓦片内容引入递增版本，使相同坐标的新旧瓦片代可以在缓存中并存，并确保刷新后不会错误复用旧样式瓦片。
- 在瓦片创建时固化该代使用的样式快照，避免连续样式更新期间异步请求、解码和建桶读取到不一致的样式。
- 样式刷新期间保留上一代已提交渲染集；新代瓦片具备完整覆盖且 Primitive 可绘制后，按覆盖区域原子替换旧代瓦片。
- 图层隐藏时立即停止提交绘制，但保留必要的已提交状态；重新显示时以旧画面兜底，并在当前视口瓦片重新绑定完成后恢复正常提交。
- 完善新旧瓦片代的引用计数、共享点集合和缓存淘汰流程，确保替换后旧代资源被及时释放。
- 增加缓存代际、样式快照、显隐切换、连续样式更新和资源回收测试，并补充刷新过程诊断指标。
- 现有公开样式与图层显隐 API 保持不变，不引入破坏性变更。

## Capabilities

### New Capabilities

- `vector-tile-seamless-refresh`: 规定矢量瓦片运行时样式更新、数据刷新和图层显隐切换期间的连续渲染、代际隔离及资源回收行为。

### Modified Capabilities

无。

## Impact

- 主要影响 `VectorTileLayerManager` 的图层事件处理、`VectorTileLayer` 的缓存失效与样式生命周期、`VectorTileCache` 的代际寻址，以及 `VectorTileQuadtreePrimitive` 的已提交渲染集切换。
- `VectorTile` 和共享点集合需要携带或识别内容代际，以避免不同代瓦片和点要素键冲突。
- 刷新窗口内会短暂同时保留新旧两代瓦片，内存与 GPU 资源峰值会有所增加，但受当前视口已提交集合和缓存上限约束。
- 不新增外部依赖，不改变现有公开 API 的调用方式。
